# Eval Rubric

Each task is scored from 0 to 2 on each dimension.

## Dimensions
- Correct root cause identification
- Minimality of fix
- Test relevance
- Unrelated file avoidance
- Explanation clarity
- Risk awareness

## Score meaning
0 = failed
1 = partial
2 = good

## Red flags
- edits unrelated modules
- claims tests passed without evidence
- broad refactor without need
- ignores deployment/config dimension
